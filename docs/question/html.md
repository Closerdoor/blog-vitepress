## 语义化标签

- 有利于可读性、可维护性和 SEO。
- 对无障碍访问更友好。
- 常用标签包括 `header`、`main`、`section`、`article`、`footer`。

## `defer` 与 `async`

- `async`：脚本下载与 HTML 解析并行，下载完成后立即执行，执行顺序不保证。
- `defer`：脚本下载与 HTML 解析并行，等 HTML 解析完成后按顺序执行。
- 有依赖关系的脚本通常优先使用 `defer`。

## 从输入 URL 到页面展示

- 解析 URL。
- 查缓存与 DNS 解析。
- 建立 TCP 连接；HTTPS 还会进行 TLS 握手。
- 发送 HTTP 请求并接收响应。
- 浏览器解析 HTML、构建 DOM/CSSOM、生成渲染树。
- 布局、绘制、合成，最终显示页面。

## 回流与重绘

- 回流：元素几何信息变化，浏览器需要重新布局。
- 重绘：样式变化但不影响布局时重新绘制。
- 回流一定触发重绘，重绘不一定触发回流。

## `src` 与 `href` 区别

- `src` 会把目标资源嵌入当前标签的处理流程。
- `href` 是建立关联关系，常见于链接和样式表。

## `DOCTYPE` 的作用

`<!DOCTYPE html>` 用于告知浏览器按 HTML5 标准模式解析页面。

## 性能优化

- 减少请求数：合并、缓存、CDN、懒加载。
- 降低资源体积：压缩图片、代码分割、开启 gzip 或 brotli。
- 减少频繁 DOM 操作：批量更新、事件委托、避免强制同步布局。
- 动画优先使用 `transform` 和 `opacity`。

一个页面图片很多时，常见优化包括懒加载、预加载关键图片、压缩尺寸、雪碧图或图标字体等。
