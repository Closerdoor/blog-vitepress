## 核心方向

- 语义化结构
- 可抓取内容
- 清晰标题与描述
- 稳定加载性能

## 语义化标签

常用标签：

- `header`
- `nav`
- `main`
- `section`
- `article`
- `aside`
- `footer`

## 标题结构

- 页面主标题通常使用一个 `h1`。
- 子模块按层级使用 `h2` 到 `h4`。
- 不要为了样式跳级使用标题标签。

## 内容与属性

- 图片应提供有意义的 `alt`。
- 页面应有 `title` 和 `description`。
- 链接文本应可读，不要全是“点击这里”。

## 技术侧注意点

- 保证首屏内容可被抓取。
- 注意 SSR、预渲染或静态站点输出。
- 减少空内容页和重复内容页。

## 补充建议

- `title`、`description` 每页尽量独立，不要整站复用同一份。
- 重要内容尽量直接输出在 HTML 中，不要完全依赖前端脚本渲染。
- 非装饰性图片补充 `alt`。
- 少用 `iframe` 承载核心内容。

## 总结

SEO 的本质不是堆关键词，而是让搜索引擎更容易理解页面主题和结构。
