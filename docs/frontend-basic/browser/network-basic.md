## 常见上网方式

### 宽带拨号

早期家庭网络常见 ADSL 或 PPPoE 拨号。

- 运营商提供宽带账号和密码。
- 电脑或路由器发起拨号。
- 拨号成功后才能访问互联网。

现在家庭场景里已经不算主流，但本质上仍然是在向运营商申请联网能力。

### DHCP 自动获取 IP

这是现在最常见的方式。

- 没有路由器时，电脑可以直接向上游设备申请地址。
- 有路由器时，通常由路由器开启 DHCP，给局域网设备分配内网 IP。

用户最常看到的效果就是“网络设置里选自动获取即可上网”。

### 静态 IP

静态 IP 有两种常见场景：

- 在局域网中手动给设备指定固定内网地址。
- 运营商直接分配固定外网 IP，这在企业专线更常见。

如果局域网没有开启 DHCP，设备往往必须手动配置 IP、子网掩码、网关和 DNS。

## 内网 IP 和外网 IP

可以把家庭网络理解成两层：

- 路由器面对运营商的一侧，通常对应一个外网 IP。
- 家里手机、电脑、平板连到路由器后，各自拿到内网 IP。

常见内网地址形态：

- `192.168.x.x`
- `10.x.x.x`
- `172.16.x.x` 到 `172.31.x.x`

几个结论：

- 局域网设备之间通过内网 IP 通信。
- 访问公网时，通常共享同一个出口外网 IP。
- 路由器承担了地址分发和转发角色。

## DNS 是做什么的

浏览器访问的是域名，但网络最终依赖 IP 通信。

DNS 的作用就是把域名解析成 IP 地址，例如：

```text
www.example.com -> 93.184.216.34
```

DNS 结果通常也会被缓存，所以并不是每次访问都重新完整查询。

## 浏览器发起请求前会做什么

一次正常页面访问，通常会经历这些基础步骤：

1. 检查本地缓存。
2. DNS 解析域名。
3. 根据协议和端口建立连接。
4. 构造请求并发送给服务端。
5. 接收响应并决定是下载、渲染还是跳转。

如果是 HTTPS，请求前还会多出 TLS 握手。

## URL 再看一遍

```text
https://example.com:443/docs/page?id=1#intro
```

- `https`：协议。
- `example.com`：主机。
- `443`：端口。
- `/docs/page`：路径。
- `id=1`：查询参数。
- `intro`：锚点。

## 请求为什么有时很快，有时很慢

常见影响因素：

- DNS 是否命中缓存。
- 是否需要新建 TCP 或 TLS 连接。
- 网络带宽和延迟。
- 浏览器缓存、CDN 缓存、服务端缓存是否命中。
- 资源体积是否过大。

## 前端更关心哪些网络缓存

- DNS 缓存：减少重复域名解析。
- CDN 缓存：让资源在离用户更近的节点返回。
- 浏览器缓存：减少重复下载。
- 服务端缓存：减少后端重复计算。

## 总结

网络基础并不复杂，重点是把几件事串起来理解：设备如何拿到 IP、浏览器如何把域名变成连接、请求如何命中缓存、资源最终如何回到页面。
